DataGate Studio release notes

Version: 17.1.19.0 - GA

Release date: 2025-12-02

The DataGate Explorer fulfills a somewhat specialized need, but when you do need it it is very handy.

When you install ASNA Visual RPG on a PC (which needs to have had Visual Studio installed prior), the ASNA DataGate Explorer implicitly installed in Visual Studio. This lets you work databases, doing such things as:

  • Create, delete, and modify DataGate Database Names
  • Add files, change files, and delete files (including logical files)
  • Browse file contents
  • Import and export database artifacts to DGIE files

Long-time ASNA users will recognize that these capabilities make the ASNA DataGate Explorer analogous to the old COM Database Manager.

The DataGate Explorer's special use case

Packaging and installing the DataGate Explorer with ASNA Visual is great for programmers, but not so much for DB administrators. For example, administrators may need to work with databases on Windows Server, but don't want to install (and license) ASNA Visual RPG. To resolve this, after installing Visual Studio on the the Windows Server, installing ASNA DataGate Studio installs the DataGate Explorer in that Visual Studio instance.

The DataGate Studio does need a license key.

Platform support

See the ASNA Version Policy for full requirements

You can only install one version of an ASNA Windows product on a single PC. For example, you can't install ASNA Visual RPG for .NET 17.0 on a PC on which ASNA Visual RPG for .NET 16.0 is installed.

General installation tips

  • We strongly recommend you apply all pending Windows updates before installing any of our Windows products.

  • Don't install any ASNA Windows products while Visual Studio is running.

  • For our products that snap into Visual Studio (ie, Visual RPG for .NET Framework, Wings, Mobile RPG, etc.) be sure to install Visual Studio first.

Downloading with Microsoft Edge

If this software is downloaded via Edge you may receive a message stating that this file "is not commonly downloaded" when attempting to install it. In this event click the View Downloads button, select the ASNA product to install from that list, and confirm that you'd like to install it.

Similarly, attempting to install the software directly through Microsoft Windows may cause a "Windows protected your PC" message to appear. In this event click the small More Info prompt on the left, and click Run Anyway on the following window.
